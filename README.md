# Family Chat Archiver

Телеграм-бот для архивирования всех сообщений семейной группы с автоматической проверкой правописания русскоязычных текстов.

## Основные возможности

✅ Сохранение всех сообщений (текст, медиа, ссылки)  
✅ Сохранение информации об авторах  
✅ Проверка орфографии через YandexSpeller API  
✅ Отправка исправлений в чат  
✅ Сохранение истории исправлений в БД  
✅ Обработка служебных событий (user_joined, title_changed и т.д.)

## Быстрый старт

### Предусловия

- MySQL 8.0+
- Python 3.8+ (для Python версии)
- Rust 1.70+ (для Rust версии)
- Telegram Bot Token (получить от @BotFather)

### Python версия

```bash
cd python

# Установка зависимостей
pip install -r requirements.txt

# Создание конфига
cp .env.example .env
# Отредактировать .env с вашими данными

# Создание БД
mysql -u root -p your_database < ../CLAUDE.md  # (нужна схема в отдельном файле)

# Запуск бота
python bot.py
```

### Rust версия

```bash
cd rust

# Создание конфига
cp .env.example .env
# Отредактировать .env с вашими данными

# Сборка
cargo build --release

# Запуск
./target/release/family-chat-archiver
```

## Конфигурация

Создайте `.env` файл в корне выбранного проекта:

```env
TELEGRAM_BOT_TOKEN=your_token_here
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_password
MYSQL_DATABASE=family_chat
LOG_LEVEL=info
```

## Схема базы данных

Все таблицы автоматически создаются при первом запуске:

- **users** — информация о пользователях
- **messages** — архив сообщений
- **media** — информация о медиа-файлах
- **links** — ссылки из сообщений
- **spelling_corrections** — история орфографических ошибок и исправлений
- **service_events** — служебные события группы

## Различия между реализациями

### Python
- **Плюсы:** Быстрая разработка, простота для тестирования
- **Минусы:** Может зависать на блокирующих операциях
- **Использование:** Разработка, testing, для маленьких групп

### Rust
- **Плюсы:** Максимальная производительность, надежность, типобезопасность
- **Минусы:** Дольше разрабатывать
- **Использование:** Production, для больших групп

## Решение проблем

### Бот не получает обновления

1. Проверьте TELEGRAM_BOT_TOKEN
2. Убедитесь, что нет других ботов с тем же токеном
3. Проверьте сетевое соединение
4. Изучите логи

### Ошибки БД

1. Убедитесь, что MySQL запущена
2. Проверьте credentials в .env
3. Убедитесь, что у пользователя есть права на создание таблиц

### YandexSpeller не работает

1. Проверьте подключение к интернету
2. YandexSpeller API может быть перегружена, есть retry logic
3. Проверьте логи для деталей ошибок

## Развертывание

Для production рекомендуется использовать Rust версию с supervisord/systemd для автозапуска.
